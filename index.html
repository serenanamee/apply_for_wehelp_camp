申請表單

個人簡介

我是 Serena，去年轉職成為 Ruby on Rails 後端工程師，工作內容主要是 api 串接，成為工程師前，主要從事和語言教學相關的工作

為什麼想成為前端、後端、或全端工程師？

進入現在的公司後，由於公司組織比較扁平化，也是草創初期的公司，老闆很 free style，因為是後端工程師，工作也主要集中在 api 串接。
關於部署方面的時候。也只有在 terminal 打上 cap production deploy 這行 code 而已， 所以對於部署的部分沒有實際操作的經驗，
如果要找下一份工作，部署這個部分是需要練習的，另外，前端的部分，也想要找時間練習，所以我是以透過上課的方式，練習及進修來參加這個 camp,
未來是以繼續當後端工程師及全端工程師邁進，現在的年薪大約在 70 萬，希望明年能夠順利換工作且年薪能夠到 80-90 萬區間。

為了成為軟體工程師，曾經做過什麼努力？

三年前，因為興趣所以在下班後利用網上免費及收費的資源學習前端技術，後來因為線上課程沒辦法讓我更有組織架構的理解到如何成為工程師，所以參加了實體全端課程

如果參與這個計畫，會怎麼安排學習時間？

公司其實工作量不會很大，所以會利用上午或是下午的時間來學習，或是下班後也會利用時間，因為有過在上班時進修的經驗，我會再開始上課後，看情況調整自身學習時間，
假日也會把平日若有落後的地方補上。

如果參與這個計畫，預期會碰到什麼困難？你打算怎麼解決它？

一直都是 JavaScript 的苦手，我可以預期在做 JS 的時候一定會撞牆，送出申請後，我會開始排進度複習 JS 及預習課程所需的內容

是否有想要加入的軟體公司？為什麼想加入該公司？

沒有特定想要加入的公司，但是目標想找到一個可以遠端工作的國際公司，希望能和來自不同國家的同事協作

想要對我們說的事情？

我知道很多工程師，能夠以自學的方式很快地，轉換語言，但我自身是屬於需要目的性的學習才能往前進，
我可以做到自己將網站開發出來，但是因為沒有動力及急迫性，可能會安於現狀的無法往前，

期待能在老師們督促下成長,進步。
